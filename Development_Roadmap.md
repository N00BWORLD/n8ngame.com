개발 로드맵: Code Chef Tycoon

이 로드맵은 '코드 쉐프 타이쿤'의 개발을 4단계(Phase)로 나누어 진행하며, 각 단계는 명확한 마일스톤과 결과물을 목표로 합니다.

Phase 1: 프로토타입 (The Proof of Concept) - 2주

목표: React 컴포넌트(에디터)와 Canvas(시뮬레이션) 간의 데이터 연동 검증 (브릿지 패턴 검증).

주요 작업:

[x] 프로젝트 환경 설정 (Vite, React, Tailwind CSS).

[ ] 기본 노드 에디터: 노드 배치(Drag & Drop) 및 와이어 연결 기능 구현 (React Flow 활용 검토 또는 커스텀).

[ ] 시뮬레이션 엔진: 노드 연결 데이터(nodes, edges)를 읽어 재료(Entity)가 이동하는 물리 로직 구현.

[ ] 상태 동기화: 에디터에서 연결을 끊으면 시뮬레이션에서도 즉시 반영되는지 검증.

결과물: 에디터에서 선을 연결하면 공이 굴러가는 형태의 최소 기능 버전 (Web).

Phase 2: 알파 버전 (Core Mechanics) - 4주

목표: 핵심 요리 루프와 기본 노드 구현을 통한 "재미" 검증.

주요 작업:

[ ] 핵심 노드 3종 구현:

공급기(Spawner): 일정 주기마다 재료 생성.

가공기(Processor): 그릴(굽기), 도마(썰기) 기능 및 소요 시간 로직.

서빙(Sink): 완성품 처리 및 코인 획득.

[ ] 타이밍 & 밸런스: 재료 이동 속도, 조리 시간, 실패(타버림) 로직 추가.

[ ] 기초 경제 시스템: 코인 획득 UI 및 노드 구매 비용 차감 연동.

[ ] 아트 리소스 적용: 임시 도형(Placeholder)을 실제 음식 및 기계 아이콘(SVG/Sprite)으로 교체.

결과물: "햄버거 만들기" 루프가 완벽하게 작동하는 단일 스테이지 빌드.

Phase 3: 베타 버전 (Content & Polish) - 6주

목표: 콘텐츠 확장, 로직 심화, 저장 기능 구현.

주요 작업:

[ ] 스테이지 확장:

Tier 1: 레모네이드 (튜토리얼/선형 구조)

Tier 2: 햄버거 (병렬 구조)

Tier 3: 피자 (루프/조건문 구조)

[ ] 고급 로직 노드:

분배기(Splitter): 재료를 여러 갈래로 나누는 라운드 로빈 로직.

검사기(Inspector): 재료 상태(익음/안 익음)에 따른 경로 분기.

[ ] 데이터 저장: localStorage 또는 Firebase를 이용한 유저 진행 상황(레벨, 돈, 배치도) 저장.

[ ] 모바일 최적화: 터치 조작감 개선, 세로 모드 UI 레이아웃 확정.

결과물: 사내 테스트 및 CBT(Close Beta Test) 가능한 수준의 완성된 빌드.

Phase 4: 정식 출시 및 라이브 (Launch)

목표: 글로벌 런칭, 수익화(BM) 탑재 및 대규모 트래픽 대응.

주요 작업:

[ ] 수익화 연동:

보상형 광고(AdMob/AdSense): 오프라인 수익 2배, 긴급 자금 지원.

인앱 결제(IAP): 젬(Gem) 구매, 프리미엄 노드, 스킨.

[ ] 분석 툴 탑재: Google Analytics, Firebase Analytics로 유저 이탈 구간 추적.

[ ] 성능 최적화:

Web Worker 도입으로 시뮬레이션 연산 분리.

오브젝트 풀링(Object Pooling)으로 메모리 누수 방지 및 가비지 컬렉션 최적화.

[ ] 튜토리얼 강화: 초기 진입 유저를 위한 인터랙티브 가이드 추가.

결과물: 앱 스토어/플레이 스토어 및 웹 플랫폼 정식 배포.
